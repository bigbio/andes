# 4-engine TMT benchmark — PXD016999 tissue proteome (2026-06-03)

Head-to-head of **Java MS-GF+**, **MSFragger**, **Sage**, and **cimas** on a
real human-tissue **TMT** dataset acquired as **ion-trap CID-MS2** (SPS-style,
low-resolution MS2) on an Orbitrap Fusion. Search parameters are harmonized
across engines and FDR is controlled uniformly with Percolator @ 1%.

> **TL;DR:** cimas returns the **most PSMs and the most distinct peptides**
> at 1% FDR, ahead of Java, MSFragger, and Sage; it ties MSFragger on proteins
> (within 0.4%). It is also the only engine that reads the Thermo `.raw` natively
> (the others run on TRFP-converted mzML).

## 1. Results

Pooled over **4 raw files — 2 per instrument** (Instrument1 `Fr01`+`Fr02`,
SecondInstrument `Fr01`+`Fr03`), uniform Percolator 3.7.1 `-Y` (TDC) at q ≤ 0.01,
identical target+decoy human FASTA and identical search parameters.

| Engine | Input | PSMs @1% | Peptides @1% | Proteins @1%* |
|---|---|---|---|---|
| **cimas** | `.raw` (native) | **22,217** | **10,473** | 4,284 |
| Java MS-GF+ | mzML | 22,029 | 10,430 | 4,159 |
| MSFragger | mzML | 20,760 | 10,093 | **4,302** |
| Sage | mzML | 19,613 | 9,644 | 4,203 |

\* Protein counts are a **rough proxy** (distinct `proteinIds` from the PSM-level
Percolator output, not parsimony / protein-level FDR). The PSM and peptide
columns are the rigorous metrics; treat the protein column as indicative.

**Relative to cimas:**

| vs | PSMs | Peptides |
|---|---|---|
| Java MS-GF+ | +0.9% | +0.4% |
| MSFragger | +7.0% | +3.8% |
| Sage | +13.3% | +8.6% |

## 2. Findings

1. **cimas leads PSMs and peptides** across both instruments; Java is a very
   close second, MSFragger and Sage trail. On proteins, MSFragger edges ahead by
   ~18 groups (0.4%) with cimas second.
2. **Native `.raw` is exclusive to cimas.** It searched the Thermo `.raw`
   directly; Java, MSFragger, and Sage required a TRFP-converted centroided mzML.
3. **Instrument-config matters for this data.** The MS2 is ion-trap CID
   (0.4 Da fragments), so all engines are configured **low-resolution / CID**
   (cimas `--instrument low-res --fragmentation CID`, Java `-m 1 -inst 0`).
   This is the same fragmentation class as PXD007683; with the correct low-res
   config cimas is competitive-to-winning here.

CID-MS2. Two physical instruments (`Instrument1`, `SecondInstrument`).

> **Note:** `SecondInstrument_…_Fr02.raw` was the originally-selected second
> SecondInstrument fraction but is replaced here by `Fr03`. `Fr02` is **not
> corrupt** — its failure was a transient `No space left on device` during mzML
> conversion (the file is large); the fix is to keep the `.raw` and the mzML on
> separate volumes so neither fills.

**FASTA:** UniProt **reviewed** (Swiss-Prot) *Homo sapiens*
([UP000005640](https://www.uniprot.org/proteomes/UP000005640)) + common
contaminants. All engines search the target-only FASTA and add reversed decoys
(Java `-tda 1`, Sage `generate_decoys`, cimas auto); MSFragger uses a
pre-built `rev_` target+decoy FASTA.

**Parameters** are taken from the project SDRF (quantms-datasets
`projects/tissue-proteomes/PXD016999`, per-instrument variant): CID dissociation,
20 ppm precursor, 0.4 Da fragment, Trypsin.

**mzML** (needed for Java/Sage/MSFragger): convert each `.raw` with
[ThermoRawFileParser](https://github.com/compomics/ThermoRawFileParser)
(`ThermoRawFileParser -i <file>.raw -b <file>.mzML -f 2 -l 2`, MS2 only).

## 4. Harmonized parameters

| Parameter | Value |
|---|---|
| Activation / instrument | **CID / low-res (ion-trap MS2)** |
| Precursor tolerance | **20 ppm** |
| Fragment tolerance | **0.4 Da** (Sage/MSFragger; Java & cimas are model-based, low-res) |
| Charge range | **2–4** |
| Enzyme / specificity | **Trypsin, fully specific (NTT = 2)** |
| Missed cleavages | **≤ 2** |
| Peptide length | **6–40** |
| Min peaks | **10** |
| Isobaric label | **TMT (+229.16293) fixed on K and peptide N-term** |
| Fixed mods | **Carbamidomethyl C (+57.02146)**, **TMT** (above) |
| Variable mods | **Oxidation M (+15.99491)**, **Acetyl protein-N-term (+42.01057)**, max 2 |
| Decoys | **reversed protein**, target-decoy |
| Threads | **8** |
| FDR | **Percolator 3.7.1 `-Y`**, q ≤ 0.01, pooled per engine over the 4 files |

Config files: [`configs/mods-tmt.txt`](configs/mods-tmt.txt) (cimas/Java
mods), [`configs/msfragger-tmt.params`](configs/msfragger-tmt.params),
[`configs/sage-tmt.json`](configs/sage-tmt.json).

### Per-engine commands (paths abbreviated)

**cimas** (native `.raw`):
```bash
cimas --spectrum file.raw --database human_sp.fasta \
  --fragmentation CID --instrument low-res --protocol TMT \
  --mods configs/mods-tmt.txt --precursor-tol-ppm 20 --precursor-cal auto \
  --isotope-error-min -1 --isotope-error-max 2 \
  --enzyme-specificity fully --max-missed-cleavages 2 \
  --min-length 6 --max-length 40 --charge-min 2 --charge-max 4 \
  --top-n 1 --min-peaks 10 --threads 8
```

**Java MS-GF+** (mzML; bigbio fork writes the Percolator PIN to `-o`):
```bash
java -Xmx7g -jar MSGFPlus.jar -s file.mzML -d human_sp.fasta -mod configs/mods-tmt.txt \
  -o java.pin -tda 1 -t 20ppm -ti -1,2 -m 1 -inst 0 -e 1 -protocol 4 -ntt 2 \
  -minLength 6 -maxLength 40 -minNumPeaks 10 -minCharge 2 -maxCharge 4 \
  -maxMissedCleavages 2 -n 1 -addFeatures 1 -thread 8
```

**MSFragger** (mzML; needs target+decoy FASTA in the params):
```bash
java -Xmx7g -jar MSFragger-4.2.jar configs/msfragger-tmt.params file.mzML
```

**Sage** (mzML):
```bash
sage configs/sage-tmt.json --write-pin -o out file.mzML
```

**Pool + Percolator** (per engine, over the 4 files):
```bash
cat <engine>.*.pin | (read -r h; echo "$h"; grep -hv "^$h") > <engine>.pooled.pin   # one header
percolator --seed 42 -Y --results-psms <engine>.t.psms.txt \
  --decoy-results-psms <engine>.d.psms.txt <engine>.pooled.pin
# PSMs/peptides @1%: rows with q-value <= 0.01 (peptides = distinct stripped sequences)
```

# 6-engine TMT benchmark — a05058 (low-res CID), uniform Percolator (2026-06-04)

Companion to the [Astral 7-engine benchmark](2026-06-04-astral-7engine.md), on a
**low-resolution ion-trap CID TMT** dataset (the opposite acquisition regime).
Same procedure: harmonized parameters, every engine re-scored through one
Percolator 3.7.1 (`-Y`, seed 42), with Java and ProSE PINs hand-built (§4 of the
Astral doc).

> **Headline (honest):** On low-res CID/TMT, cimas is **mid-pack on yield** while
> remaining the **fastest** real search. This is the known low-res-CID gap — the
> GF-free penalty plus the CID node-scoring divergence that **Phase 3 (own CID
> models)** targets — not a regression. Contrast with high-res Astral, where cimas
> leads.

## 3. Findings

1. **cimas trails the leaders on low-res yield** (proteins/PSMs/peptides) but is the
   fastest real search (1:08). The chimeric cascade barely moves low-res TMT
   (10,453 vs 10,417 PSMs) — co-isolation recovery is a high-res phenomenon here.
   This matches the documented "low-res CID ~6%" gap; the root cause is CID
   node-scoring, addressed by Phase 3 (own models), not calibration.
2. **MSFragger leads** on all three count metrics; Java MS-GF+ is a close second on
   proteins/peptides (and far faster here — 9:50 — than on high-res Astral, where it
   took 2 h, because low-res CID has a much smaller fragment search space).
3. **ProSE is the wrong tool for low-res data.** It is a high-res fragment-index
   engine that **caps fragment tolerance at 0.1 Da**; forced to 0.1 Da on ion-trap
   CID (where fragment peaks scatter ±0.3–0.5 Da) it matches far fewer fragments,
   hence the lowest counts. Reported for completeness with this caveat.

## 4. Procedure

Same as the Astral doc §4, with low-res settings:
- **cimas** (`tmt_all5.sh`): `--fragmentation CID --instrument low-res --protocol TMT`.
- **Sage** (`configs/sage-a05058.json`): low-res `fragment_tol.da [-0.4,0.4]`.
- **MSFragger** (`configs/msfragger-a05058.params`): `fragment_mass_tolerance = 0.4`,
  `fragment_mass_units = 0` (Da), `data_type = 0`.
- **Comet** (`comet_tmt.sh`): low-res binning `fragment_bin_tol = 1.0005`,
  `fragment_bin_offset = 0.4`, `theoretical_fragment_ions = 1`; TMT static mods.
- **Java MS-GF+**: `-inst 0 (low-res) -m 1 (CID) -protocol 4 (TMT)`. mzid →
  `MzIDToTsv` → `build_pins.py java` → Percolator (msgf2pin crashes on the mzid).
- **ProSE** (`tmt_finalize.sh`): fragment tol pinned to its 0.1 Da maximum;
  idXML annotations → `build_pins.py prose` → Percolator.

## 5. Reproduce

```bash
bash scripts/tmt_all5.sh        # cimas, Java, Sage, MSFragger, ProSE
bash scripts/comet_tmt.sh       # Comet (low-res TMT params)
bash scripts/tmt_finalize.sh    # ProSE@0.1Da re-run, build Java+ProSE PINs, percolate all 6
```

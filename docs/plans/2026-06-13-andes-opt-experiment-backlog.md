# andes Optimization — Ranked Experiment Backlog (`/loop` round-robin)

> Companion to `2026-06-13-andes-search-engine-optimization-campaign.md`. That doc is the standing
> spec (objective, harness, guardrails, log). This doc is the **ranked, ready-to-pick backlog**:
> one atomic, FDP-gated, byte-identical-when-off experiment per row, interleaved across the three
> axes (PSMs / SPEED / PARAMS) so the loop can round-robin.
>
> **Pick order = top of table down.** Skip a row only if a higher row is blocked (compute pending,
> dependency unlanded). Every row is FDP-gated and byte-identical when off by construction. Known
> dead ends (chimeric-TMT, fragmentation overlay, rank-model ceiling via data/recal, fragment-index
> speed-v2) are excluded — none of the rows below re-propose them.

## The first 3 to run (and why)

1. **#1 — PSMs: Auto-detect isobaric protocol from MS2 reporter cluster** *(PARAMS+PSMs, high, small).*
   This is the campaign heart (**win TMT**) AND a param-shed in one shot. It does NOT need Codon — it
   unlocks the **already-validated +~3.5% TMT gain** (the windowed dense-peak filter) that is today
   silently locked behind a manual `--protocol TMT`. Smallest effort × highest leverage, runnable
   immediately on the VM, and byte-identical on Astral because the detector cannot fire on label-free
   data. Run this **first**.

2. **#2 — SPEED: single-segment fast path in the node-score cache fill** *(SPEED, high, small).*
   Pure byte-identical hot-path win (removes ~N× redundant `segment_num` work in the per-spectrum
   cache fill, which dominates `ScoredSpectrum::new`). Provably a no-op transform for `num_segments==1`
   models (the Astral/high-res canary). Independent of #1, small, and proves the SPEED axis early with
   zero FDP risk. Run **second** so the loop has a speed win banked while the TMT model work spins up.

3. **#3 — PSMs: Train a dedicated low-res CID-TMT model** *(PSMs, high, large).*
   The **structural** root cause of the TMT gap: there is literally no `(CID, LowRes, tmt)` model — a
   low-res TMT search falls back to `cid_lowres_tryp` (Standard, esf=0, unlabeled-tryptic tables). This
   is the highest-ceiling PSM lever, but it is **large** (Codon: MSFragger→Percolator gold-PSM
   generation on a public CID-TMT corpus → VM: `andes train` on those labels) so it is kicked off
   third in parallel with the cheap wins above; experiments #6/#9/#11 are sweeps that ride on the
   seed this produces. Start the Codon gold-PSM generation as soon as #1 confirms TMT is being
   routed correctly.

**Shape:** #1 unlocks the existing TMT gain for free; #2 banks a no-risk speed win; #3 is the
long-pole model that lifts the TMT ceiling and feeds the four sweeps (#6, #9, #11, #13). Cheap
byte-identical speed/param wins (#2, #4, #5, #7, #8) keep the loop productive while #3's gold PSMs
are generated on Codon and andes is trained on the VM. Additive Percolator features (#10, #12) and
the speculative param/measure-first items (#13, #14, #15) tail the list.

## Notes for the loop

- **Round-robin cadence:** the table already interleaves axes (PARAMS/PSMs → SPEED → PSMs → SPEED → SPEED
  → PSMs → SPEED → PARAMS → PSMs → …). Take rows top-down; if the next row is blocked (#3 corpus pending,
  #15 dependency unlanded), drop to the next runnable row of a different axis.
- **#6 / #9 / #11 ride on #3's seed** — they are model sweeps, not independent builds; don't start them
  before a CID-TMT seed + corpus exists, but they are cheap once it does.
- **Astral is always the canary**, even for TMT-aimed rows: every PSM/model row above asserts Astral
  byte-identical or no-regression. Grep the resolver's `Param model:` line for Astral before comparing
  counts (a new model row must never change Astral's `model_id`).
- **Dropped as known dead ends (not in table):** chimeric-TMT, fragmentation overlay, rank-model ceiling
  via data/recal, fragment-index speed-v2. None of the mined experiments re-proposed these, so nothing
  was cut — but the loop must still refuse them if they resurface.
- **Verdicts go in the campaign doc's Experiment Log**, not here; this backlog is the pick-list only.

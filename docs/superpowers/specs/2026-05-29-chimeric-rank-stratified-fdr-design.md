# Chimeric rank-stratified FDR — measure-first design

**Date:** 2026-05-29
**Branch:** `feat/chimeric-dda-plus`
**Type:** Measurement / analysis (no Rust production change)
**Follows:** `2026-05-29-chimeric-phase3-bench-canary-fails.md` (Phase 3 refuted —
per-PSM rescore deflates targets+decoys symmetrically, so pooled PSM-level FDR
doesn't move). This tests the *structural* fix the Phase-2/3 post-mortems pointed
to: a per-scan / rank-level FDR model.

## Goal & decision rule

Test whether **rank-stratified separate FDR** makes a chimeric search trustworthy:
the Astral canary returns toward ~36.7k while PXD beats Java. Measure-first on the
existing chimeric PIN — **no Rust production change**.

**Decision rule:**
- Canary passes (Astral total ≈ 36.7k) **and** PXD total > Java 14,974 →
  model validated → open a follow-up to productionize (where the rank-aware FDR
  lives permanently).
- Astral still inflated → rank-stratification insufficient → rethink (scan-level
  grouping, or a stricter rank≥2 null) before any build.

## Model (Approach A — rank-stratified separate FDR)

Pooled PSM-level TDC over ~5 PSMs/scan fails because it mixes **rank-1** (clean,
one real ID/scan) with **rank≥2** (mostly spurious co-emissions) in one q-curve;
the rank≥2 junk decoys inflate the whole set. Fix: compute target-decoy FDR
**separately within each stratum**.

- **rank-1 stratum** = best peptide per scan ≈ the Java-comparable single-ID set.
  On Astral, rank-1@1% ≈ off baseline (~36.7k) — selection is ~unchanged.
- **rank≥2 stratum** = additional co-isolated peptides. On narrow windows (Astral)
  rank≥2 targets *and* decoys are both ~random → almost nothing clears 1% → ~0.
  On wide windows (PXD) it admits *real* co-isolated peptides at controlled error.
- **Total IDs = rank-1@1% + rank≥2@1%.** rank-1@1% is the number compared to Java;
  total is rank-1 + the trustworthy rank≥2 bonus.

The canary passes **by construction** for the primary set, and the bonus is only
non-trivial where co-isolation is real.

## Measurement (Percolator-native, per stratum)

1. **Rank extraction.** From PIN `SpecId = {spec_id}_{scan}_{rank}_{rowidx}`,
   recover `rank` via `rsplit('_')` (`rowidx`=last, `rank`=2nd-last, `scan`=3rd-last
   — robust to underscores/spaces inside `spec_id`).
2. **Split** the PIN into `rank1.pin` (rank==1) and `rankN.pin` (rank≥2), each
   preserving the header.
3. **Run Percolator separately** on each (per-stratum SVM training = the
   principled "separate FDR"; decoy symmetry automatic via Percolator labels).
4. **Count** targets@1% FDR in each; **total = rank-1@1% + rank≥2@1%**.

Run on **both** PIN variants to separate effects:
- **(b1) Non-rescored Phase-1 PIN** (multi-emission, residual rescore skipped):
  isolates the FDR-*model* effect. Generated by the same binary with the rescore
  bypassed (env gate `MSGF_CHIMERIC_NO_RESCORE=1`, chimeric-only, test-path).
- **(b2) Phase-3 rescored PIN** (already produced): shows whether the residual
  rescore + rank-stratified FDR *compose* (rescore further deflates the rank≥2
  stratum).

## Validation gates

| Gate | Expectation |
|---|---|
| **Canary (decisive)** | Astral total ≈ 36.7k (rank-1 ≈ 36.7k, rank≥2 ≈ ~0) |
| **Sanity** | rank-1-only ≈ the `off` baseline per dataset (validates the split) |
| **Win** | PXD total > Java 14,974, rank≥2 contributing trustworthy extras |
| **Decoy balance** | restored toward healthy within each stratum |
| TMT (context) | not a target; report for completeness |

## Deliverable

- `benchmark/parity/rank_stratified_fdr.{py,sh}`: split-by-rank + 2 Percolator
  runs per dataset + summary table (off / pooled-on / rank-1@1% / rank≥2@1% /
  total, per dataset, for both PIN variants).
- A results note `docs/parity-analysis/notes/2026-05-29-rank-stratified-fdr-bench.md`.
- **No change to any production Rust path.** The only Rust touch is the test-only
  `MSGF_CHIMERIC_NO_RESCORE` env gate (chimeric-only) to regenerate the Phase-1 PIN.

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| rank-1-only ≠ off (chimeric wide window changed top-1 for some scans) | Expected/​informative; report the delta; off remains the reference |
| Percolator can't train on junk-heavy rank≥2 stratum | That's the desired outcome (≈0 passes); confirm via decoy fraction |
| Pre-Phase-3 PIN unavailable on VM | Regenerate with `MSGF_CHIMERIC_NO_RESCORE=1` |
| rank parse wrong if SpecId format differs | `rsplit('_')` + assert numeric rank/scan; sanity-check row counts vs source PIN |

## Out of scope

- Productionizing the rank-aware FDR (separate follow-up if validated).
- Changing the merge-gate definition (this is measurement; gate unchanged).
- TMT (chimeric net-negative there); GF-shape work (Lever 2a).
